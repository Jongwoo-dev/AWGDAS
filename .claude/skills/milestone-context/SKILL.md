# /milestone-context — 마일스톤 컨텍스트 스냅샷

마일스톤 작업 간 세션을 clear하고 새로 시작할 때, 이전 컨텍스트를 유지하기 위한 스킬.

## 사용법

```
/milestone-context M0
```

인자로 마일스톤 이름(예: `M0`, `M1`, …)을 받는다.

---

## 동작 규칙

### 1. 컨텍스트 스냅샷 생성/갱신

인자로 받은 마일스톤 이름에 해당하는 `docs/context/{마일스톤}.md` 파일을 생성 또는 갱신한다.

**Append 방식**: 새 스냅샷을 파일 **맨 위에** 추가하고, 기존 내용은 `---` 구분선 아래에 보존한다.

스냅샷에 반드시 포함할 내용:

1. **타임스탬프** — 스냅샷 생성 시각 (예: `2026-03-02 14:30`)
2. **프로젝트 구조** — `src/` 디렉토리 트리 (`tree` 또는 `find` 로 생성)
3. **이번 마일스톤에서 생성/수정한 주요 파일 목록** — git diff/log 기반으로 파악
4. **아키텍처 상태 요약**:
   - 현재 동작하는 것
   - 아직 구현되지 않은 것
5. **다음 마일스톤 안내** — `docs/milestones.md` 참조하여 다음 마일스톤 이름과 핵심 목표 기재

### 2. 컨텍스트 복원 (다음 마일스톤 시작 시)

다음 마일스톤을 시작할 때, 직전 마일스톤의 context 파일(`docs/context/{직전마일스톤}.md`)을 읽어 현재 프로젝트 상태를 파악한 후 작업을 시작한다.

---

## 실행 절차

아래 절차를 순서대로 수행하라:

1. **인자 파싱**: 첫 번째 인자를 마일스톤 이름으로 사용 (예: `M0`)
2. **milestones.md 읽기**: `docs/milestones.md`를 읽어 해당 마일스톤의 상태와 다음 마일스톤 정보 파악
3. **프로젝트 구조 수집**: `src/` 디렉토리 트리를 수집
4. **변경 파일 파악**: git log/diff로 이번 마일스톤에서 변경된 파일 목록 수집
5. **기존 context 파일 확인**: `docs/context/{마일스톤}.md` 가 이미 존재하면 내용을 읽어둠
6. **스냅샷 작성**: 아래 포맷으로 스냅샷을 생성
7. **파일 쓰기**: 새 스냅샷 + `---` 구분선 + 기존 내용 순서로 파일에 쓰기
8. **완료 보고**: 생성된 파일 경로와 요약을 사용자에게 출력

---

## 스냅샷 포맷

```markdown
# {마일스톤} 컨텍스트 스냅샷

> 생성: {YYYY-MM-DD HH:mm}

## 프로젝트 구조 (src/)

\```
{src/ 트리 출력}
\```

## 이번 마일스톤 변경 파일

{생성/수정된 주요 파일 목록 — bullet list}

## 아키텍처 상태

### 동작하는 것
{현재 구현 완료된 기능 bullet list}

### 아직 없는 것
{미구현 기능 bullet list}

## 다음 마일스톤

- **{다음 마일스톤 ID}**: {다음 마일스톤 제목}
- 핵심 목표: {1~2줄 요약}
- 세부 문서: `docs/milestones/{다음 마일스톤 파일}.md`
```

---

## 주의사항

- `docs/context/` 디렉토리가 없으면 자동 생성한다
- 기존 스냅샷이 있으면 덮어쓰지 않고 맨 위에 prepend한다
- 마일스톤 이름이 `docs/milestones.md`에 없으면 사용자에게 경고를 출력한다
