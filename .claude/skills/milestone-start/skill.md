# /milestone-start — 마일스톤 작업 시작

이전 마일스톤 컨텍스트를 복원하고 새 마일스톤 작업을 시작한다.

## 사용법

```
/milestone-start M3
```

인자로 시작할 마일스톤 이름(예: `M3`)을 받는다.

---

## 실행 절차

아래 절차를 순서대로 수행하라:

1. **인자 파싱**: 첫 번째 인자를 마일스톤 이름으로 사용 (예: `M3`)
   - 인자가 없으면 사용자에게 마일스톤 이름을 되물어라

2. **milestones.md 읽기**: `docs/milestones.md`를 읽어 다음을 파악:
   - 해당 마일스톤의 존재 여부 (없으면 경고 후 되물어라)
   - 선행 의존 마일스톤 목록 (예: M3의 의존은 M1)
   - 세부 문서 경로

3. **직전 컨텍스트 복원**: `docs/context/` 디렉토리에서 대상 마일스톤 직전 번호의 컨텍스트 파일 1개를 읽는다
   - 예: M3 시작 시 → `docs/context/M2.md` 읽기
   - 직전 파일이 없으면 그 이전 파일을 탐색 (M2 없으면 M1 시도)
   - 컨텍스트 파일이 하나도 없으면 경고를 출력하되 계속 진행

4. **마일스톤 세부 문서 읽기**: `docs/milestones/` 에서 대상 마일스톤의 세부 문서를 읽는다
   - 세부 문서가 없으면 경고를 출력하되 계속 진행

5. **시스템 설계 문서 읽기**: `docs/system-design.md`를 읽는다

6. **현재 상태 요약**: 사용자에게 다음을 출력:
   - 복원된 컨텍스트 요약 (이전 마일스톤에서 구현된 것)
   - 이번 마일스톤 목표 및 작업 목록
   - 필요한 선행 의존이 충족되었는지 여부

7. **작업 시작**: plan mode로 진입하여 마일스톤 구현 계획을 수립한다

---

## 주의사항

- 인자가 없거나 유효하지 않으면 반드시 사용자에게 되물어라. 추측하지 마라.
- 컨텍스트 파일 읽기 실패는 치명적이지 않다. 경고만 출력하고 진행한다.
- 이 스킬은 읽기 + 계획 수립만 수행한다. 코드 수정은 plan mode 승인 후에 한다.
