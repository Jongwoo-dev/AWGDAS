## 아키텍처 상태

### 동작하는 것
- TypeScript strict mode + ESM (NodeNext) 빌드 파이프라인
- `npm run typecheck` → 에러 0
- `npm run build` → `dist/` 정상 생성
- `npm test` → 88 tests 전체 PASS
- 공유 타입 정의 (M1): 리터럴 유니온 4개, 에이전트 간 전달 인터페이스 8개, 상태/설정 4개
- 유틸리티 (M2):
  - `logger`: createLogger + child() 패턴, 로그 레벨 필터링 (AWGDAS_LOG_LEVEL)
  - `fileManager`: ensureOutputDir, writeGameFile, readGameFile, deleteGameFile, gameFileExists — path traversal 방지, baseDir DI
  - `manifest`: createManifest, readManifest, addFileToManifest, updateFileInManifest, removeFileFromManifest — 섹션 18 준수
  - `anthropicClient`: getClient (싱글톤), callAgent (텍스트 전용), callAgentWithTools (tool_use) — 3회 재시도 + exponential backoff (1s/2s/4s), role별 maxTokens/timeout
- 상태 머신 (M3):
  - `roundStateMachine`: createRoundState, transition, canRetry, incrementRetry, 데이터 슬롯 setter
  - 불변 상태 관리 (스프레드 연산자), 유효 전환 맵 기반
- 에이전트 프롬프트 (M4):
  - `PL_SYSTEM_PROMPT`: Round Spec 생성용 — 역할, 금지사항, Scope Lock/Retry/AC 정책, RoundSpec JSON 스키마
  - `PLANNER_SYSTEM_PROMPT`: 기능 분해용 — 역할, 금지사항, 파일분리원칙, FeatureBreakdown JSON 스키마
  - `DEVELOPER_SYSTEM_PROMPT`: 기능 구현용 — 역할, 금지사항, manifest 규칙, tool_use 지시, DevResult JSON 스키마
  - `QA_SYSTEM_PROMPT`: AC 검증용 — 역할, 금지사항, 무결성 검증, DoD 6개 조건, QAResult JSON 스키마
  - barrel export로 `import { XX_SYSTEM_PROMPT } from "./prompts/index.js"` 패턴 사용 가능
