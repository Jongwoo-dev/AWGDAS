## 아키텍처 상태

### 동작하는 것
- TypeScript strict mode + ESM (NodeNext) 빌드 파이프라인
- `npm run typecheck` → 에러 0
- `npm run build` → `dist/` 정상 생성
- `npm test` → 88 tests 전체 PASS
- 공유 타입 정의 (M1): 리터럴 유니온 4개, 에이전트 간 전달 인터페이스 8개, 상태/설정 4개
- 유틸리티 (M2): logger, fileManager, manifest, anthropicClient (3회 재시도 + exponential backoff, streaming 방식)
- 상태 머신 (M3): createRoundState, transition, canRetry, incrementRetry, 데이터 슬롯 setter — 불변 상태 관리
- 에이전트 프롬프트 (M4): PL, Planner, Developer, QA 4종 system prompt + barrel export
- PL 에이전트 (M5): `runPLInit` — callAgent로 PL API 호출, parseJsonResponse로 RoundSpec 파싱
- Planner 에이전트 (M5): `runPlanner` — callAgent로 Planner API 호출, parseJsonResponse로 FeatureBreakdown 파싱
- JSON 파서 (M5): `parseJsonResponse` — 직접 파싱 + markdown fence 제거 fallback
- Developer 에이전트 (M6): `runDeveloper` — callAgentWithTools로 tool_use 루프, write_file/read_file/delete_file 3종 tool, fileManager I/O + manifest 즉시 갱신, parseJsonResponse로 DevResult 파싱
- 파이프라인 (M6): stdin → PL_INIT → PLANNER_DEFINE → DEV_IMPLEMENT → 결과 출력 → 종료. dotenv로 .env 환경변수 로딩, slugify로 gameName 생성, output/{gameName}/ 디렉토리에 게임 파일 생성
