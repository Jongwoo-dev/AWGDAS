## 이번 마일스톤 변경 파일

- `src/agents/plAgent.ts` — 신규 생성. `runPLInit(gameDescription, roundId): Promise<RoundSpec>` — PL API 호출 → RoundSpec JSON 파싱
- `src/agents/plannerAgent.ts` — 신규 생성. `runPlanner(spec: RoundSpec): Promise<FeatureBreakdown>` — Planner API 호출 → FeatureBreakdown JSON 파싱
- `src/utils/parseJson.ts` — 신규 생성. `parseJsonResponse<T>(text, label): T` — JSON.parse + markdown fence 제거 fallback
- `src/index.ts` — 수정. stub → 부분 파이프라인 (stdin → PL_INIT → PLANNER_DEFINE → 결과 출력)
- `.env.example` — 신규 생성. 환경변수 샘플 (ANTHROPIC_API_KEY, AWGDAS_MODEL, AWGDAS_LOG_LEVEL)
- `package.json` / `package-lock.json` — dotenv 의존성 추가

## 아키텍처 상태

### 동작하는 것
- TypeScript strict mode + ESM (NodeNext) 빌드 파이프라인
- `npm run typecheck` → 에러 0
- `npm run build` → `dist/` 정상 생성
- `npm test` → 88 tests 전체 PASS
- 공유 타입 정의 (M1): 리터럴 유니온 4개, 에이전트 간 전달 인터페이스 8개, 상태/설정 4개
- 유틸리티 (M2): logger, fileManager, manifest, anthropicClient (3회 재시도 + exponential backoff)
- 상태 머신 (M3): createRoundState, transition, canRetry, incrementRetry, 데이터 슬롯 setter — 불변 상태 관리
- 에이전트 프롬프트 (M4): PL, Planner, Developer, QA 4종 system prompt + barrel export
- PL 에이전트 (M5): `runPLInit` — callAgent로 PL API 호출, parseJsonResponse로 RoundSpec 파싱
- Planner 에이전트 (M5): `runPlanner` — callAgent로 Planner API 호출, parseJsonResponse로 FeatureBreakdown 파싱
- JSON 파서 (M5): `parseJsonResponse` — 직접 파싱 + markdown fence 제거 fallback
- 부분 파이프라인 (M5): `index.ts`에서 stdin → PL_INIT → PLANNER_DEFINE → 결과 출력 → 종료. dotenv로 .env 환경변수 로딩
- 각 단계별 상태 전환 로깅, 전체 try/catch 에러 처리 (exit code 0/1)
